# BeyVox Server (Сервер сообщества)

Бэкенд-сервер инстанса (сообщества) BeyVox. Написан на языке **Rust** с использованием веб-фреймворка **Axum**. Хранит текстовые сообщения, структуру каналов и интегрируется с **LiveKit SFU** для передачи голосового трафика в реальном времени.

## Стек технологий
- **Язык**: Rust
- **Веб-фреймворк**: Axum + Tokio
- **База данных**: PostgreSQL (миграции через SQLx)
- **Голосовая связь**: LiveKit Server (WebRTC SFU)
- **Контейнеризация**: Docker & Docker Compose

## Быстрый запуск с помощью Docker

Самый простой способ запустить сервер — использовать Docker Compose. Проект включает в себя преднастроенный стек: бэкенд на Rust, базу данных PostgreSQL и сервер LiveKit.

1. **Подготовка ключей LiveKit**:
   Убедитесь, что в папке `server` есть файл `livekit-keys.yaml` (можно скопировать из `livekit-keys.example.yaml`):
   ```yaml
   keys:
     devkey: devsecret_at_least_32_chars_long!!
   ```

2. **Настройка окружения**:
   Скопируйте пример файла конфигурации:
   ```bash
   cp .env.example .env
   ```
   Отредактируйте `.env` и настройте параметры (название сервера, секретные ключи, пароль БД).

3. **Запуск**:
   Запустите все сервисы в фоновом режиме:
   ```bash
   docker compose up -d --build
   ```
   После этого сервер будет доступен на порту `8080` (вы можете подключиться к нему из клиента по адресу `http://localhost:8080` или вашему домену).

## Локальная разработка без Docker

Если вы хотите запустить проект локально для отладки кода:

1. Убедитесь, что у вас установлен Rust (версия 1.75+) и запущены PostgreSQL (на порту `5433` или настроенном в вашей строке подключения) и LiveKit.
2. Примените миграции базы данных или настройте схему.
3. Установите переменные окружения, соответствующие вашему окружению.
4. Запустите проект:
   ```bash
   cargo run
   ```
