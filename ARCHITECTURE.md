# BeyVox — Архитектурные решения

Документ фиксирует все решения принятые в ходе проектирования.
Обновляется по мере изменения решений.

---

## Концепция

BeyVox — кросс-платформенный голосовой мессенджер (аналог Discord/TeamSpeak).
Модель: self-hosted серверы + центральный каталог.

- Каждый желающий поднимает свой сервер
- Центральный сервис на `beyvox.beykus.fun` хранит каталог публичных серверов (`name, ip/domain, description, icon, online_count, tags`)
- Приватные серверы не регистрируются в каталоге — подключение по прямому IP/домену
- Центральная авторизация: пользователь регистрируется один раз, аккаунт работает на любом сервере

## Стек

| Компонент | Технология |
|---|---|
| Бэкенд сервера + центральный каталог | Rust + Axum |
| БД сервера | PostgreSQL |
| БД клиента | SQLite |
| SFU (голос + стриминг) | LiveKit (self-hosted) |
| Клиент — оболочка | Tauri |
| Клиент — UI | Vue 3 + TypeScript + Vite |
| Аудио I/O | cpal |
| VST хостинг | VST2 (`.dll`/`.so`/`.dylib`) + VST3 (`.vst3`) |
| Аудио кодек | OPUS 48kHz |
| Видеокодек (стриминг) | H264 (основной) + VP8 (фолбэк) |
| Шифрование соединений | TLS 1.3 |
| Шифрование медиа | SRTP |
| Аутентификация | JWT (access 15 мин + refresh rotation) |
| Хэширование паролей | argon2 |

## Авторизация и федерация

- Пользователь логинится на `beyvox.beykus.fun`, получает JWT
- Self-hosted серверы верифицируют JWT **локально** через JWKS endpoint:
  `beyvox.beykus.fun/.well-known/jwks.json`
- Сервер кэширует публичный ключ — не зависит от доступности центрального сервиса
- Если центральный сервер лежит — уже залогиненные пользователи продолжают работать

## Голос и медиа

- Транспорт: WebRTC + OPUS через LiveKit SFU
- PTT (Push-to-Talk) и VAD (Voice Activity Detection) — выбор пользователя
- **Simulcast**: стример отдаёт несколько качеств (1080p/720p/360p), LiveKit форвардит нужное каждому зрителю в зависимости от канала
- Стриминг экрана: захват всего экрана или конкретного окна, H264 + simulcast
- E2EE (шифрование с per-user ключами) — отложено на будущее, SRTP достаточно для MVP

---

## VST плагины

- Пользователь загружает `.dll` / `.so` / `.vst3` файл в настройках клиента
- Плагин привязан к клиенту, применяется к аудиопотоку с микрофона до OPUS энкодинга
- Поддерживаемые форматы: **VST2** (`.dll`/`.so`/`.dylib`) и **VST3** (`.vst3`)
- UI плагина открывается в **нативном окне** (HWND/NSView/XWindow) — стандартное поведение DAW
- Настройки плагина сохраняются в SQLite клиента
- В MVP можно начать с VST2, VST3 добавить следующим этапом

## Структура репозиториев

```
beyvox-server/     — self-hosted сервер (Rust + Axum + PostgreSQL)
beyvox-central/    — центральный каталог + auth (Rust + Axum)
beyvox-client/     — Tauri + Vue 3 клиент
beyvox-proto/      — общие схемы сообщений (MessagePack/JSON)
```

---

## Протокол

- **WebSocket** — управляющий трафик (signaling, чат, присутствие)
- **WebRTC/UDP** — медиапотоки через LiveKit
- **REST API** — auth, CRUD серверов/каналов/сообщений
- Формат сообщений: JSON (MVP), MessagePack (оптимизация позже)

Opcodes WebSocket: `HELLO`, `IDENTIFY`, `READY`, `MESSAGE_CREATE`,
`VOICE_STATE_UPDATE`, `VOICE_SERVER_UPDATE`, `PRESENCE_UPDATE`, `HEARTBEAT`

---

## Что НЕ входит в v1.0

- Видеозвонки (только голос + стриминг экрана)
- DM между пользователями
- Мобильный и браузерный клиент
- Боты и сторонний API
- Монетизация / подписки
- E2EE голоса (SRTP достаточно для v1.0)
